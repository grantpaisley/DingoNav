Architecture

Connection flow

  1. requestDevice filtered on service 6000… (namePrefix fallback) → Android shows the per-VIN PIN bond dialog once.
  2. gatt.connect() → getPrimaryService → getCharacteristic.
  3. startNotifications() only — never readValue() (returns nothing on this VCU).
  4. Decode notifications → HUD. First 5 raw frames logged to console to pin endianness/scale.
  5. Reconnects use getDevices() — bonded bike, no picker, no PIN. Drops auto-retry with backoff (1→15 s).
